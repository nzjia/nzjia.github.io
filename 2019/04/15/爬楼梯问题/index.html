可以看出 $Cn > 2^{n/2}$ ，意味着 climb_stairs() 将是 n 的指数级。

这个函数不好的原因是第 n 阶楼梯取决于前两个的值，即 $F_{n-1}$ 和 $F_{n-2}$ 。但是在计算 $F_{n-2}$ 之后，因为它不知道前面已经被计算出来的 $F_{n-2}$ 的值，导致了重复的操作，正因为这种滚雪球的效应导致了指数倍的运行时间。

使用线性递归解决问题

我们可以更有效地使用递归来计算 $F_n$，重新定义函数的期望值，我们定义一个递归函数，该函数返回一个连续的斐波那契数列 ($Fn$, $F_{n-1}$)，并且约定 $F_{n-1}$ = 0，而不是返回在第 n 个台阶时单一的数值。这样似乎是增加了负担，但从递归的角度来说，通过这个额外的信息可以是的递归更容易继续进行，避免重复计算。这样子整体的运算时间在 O(n) 的时间内完成。

通过字典（散列表）等数据结构保存状态

我们可以通过一个数据结构（比如散列表）来保存已经求解过的 $F_n​$，当递归调用到 $F_n​$ 的时候，先查看是否已求解过了。如果是，则直接从数据结构中取值返回，不需要重复计算。
